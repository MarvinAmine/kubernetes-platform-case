Stage 2 environment architecture

Four environment views, one governed promotion model

Stage 2 uses local, dev, staging, and prod as real environments. Dev and staging share the non-prod AKS platform cluster, while production is isolated in its own AKS cluster.

Local

Fast developer validation

Local proves the application, Helm chart, runtime configuration, and basic observability without depending on Azure.
